# ValuCast UI Pass Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans (or subagent-driven-development) to implement task-by-task. Steps use checkbox (`- [ ]`) syntax. Five staged groups (A–E) in the spec's foundation-first order; each group is independently shippable. Checkpoint after each group.

**Goal:** Split horizon from scoring, pin a consolidated toolbar, finish the accessibility pattern, tokenize + lift legibility, and turn the methodology page into a self-substantiating model explainer with a drift-locked validation scorecard.

**Architecture:** Horizon tabs become navigation links (`mode` derived → active tab); scoring stays `mode` radios re-rendered via htmx; horizon change = full navigation, in-horizon = partial swap. A measured `--toolbar-h` CSS var keeps the sticky table header below the wrapping toolbar. The methodology page renders validation numbers from a committed `methodology_scorecard.json` (produced by the rolling-origin harnesses), drift-locked by tests against the artifact and the params modules.

**Tech Stack:** Flask, Jinja2, htmx, vanilla CSS/JS, Python `unittest` (pytest).

**Spec:** `docs/superpowers/specs/2026-06-09-ui-pass-design.md`

**Test command:** `python -m pytest tests -q -p no:cacheprovider` (subsets per task).

```python
class TestTypeScale(unittest.TestCase):
    def setUp(self):
        self.client = app.test_client()

    def test_no_content_text_below_floor(self):
        import re
        css = self.client.get("/static/style.css").data.decode("utf-8")
        # No content rule may use < 0.8rem. Decorative micro-labels are tagged
        # with the marker comment /* micro */ on the same line and are exempt.
        offenders = []
        for line in css.splitlines():
            m = re.search(r"font-size:\s*0\.(6\d|7\d)rem", line)
            if m and "/* micro */" not in line:
                offenders.append(line.strip())
        self.assertEqual(offenders, [], f"sub-0.8rem content sizes: {offenders}")
```
